## Gotchas

- **zustand v5 selector trap (cost us a crash this session):** NEVER return a freshly-built array/object from a `useGameStore(s => ...)` selector (e.g. `s => s.state.creatures.filter(...)`). zustand v5 uses `useSyncExternalStore`; a new reference every render reads as a changed snapshot ‚Üí infinite loop ‚Üí "getSnapshot should be cached" ‚Üí component crashes ON MOUNT. Subscribe to the stable slice (`s => s.state` or `s => s.state.creatures`) and do `.filter`/`.find`/`.map` in the render body. **Two static code reviews missed this; only the live browser mount caught it ‚Äî always do a real mount/QA drive on new RN screens (no component-test coverage exists).**
- **Store hydration is root-level and persistence is gated on `loaded` (fixed in Plan 4, don't regress):** `load()` runs in `app/_layout.tsx`, NOT per-screen. Before the fix, deep-linking straight to /town ran commits on the fresh initial state and PERSISTED it ‚Äî silently wiping the real save. Any new screen with mutating actions needs no load call of its own; any new store commit path must go through `commit`/`commitWithDiscovery` (they skip `persist` until `loaded`). Two static reviews + all unit tests missed this; only the live direct-URL reload caught it ‚Äî keep "hard reload on every tab" in QA drives.
- **Expo in a git worktree needs node_modules:** worktrees don't share the main checkout's `node_modules`; `npx expo start` half-works and serves 404 bundles (Metro error JSON with MIME `application/json`).
