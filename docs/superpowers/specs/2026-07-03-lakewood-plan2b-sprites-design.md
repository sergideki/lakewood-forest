# Lakewood — Plan 2b: Creature sprites (design)

Replace the emoji creature placeholders with an original set of ~10 Gen-4-style 64×64 pixel
critters, via a localized, fallback-safe swap. Follows Plan 2 (Forest & creatures), which shipped
on emoji.

**Decided 2026-07-03:**
- Art source: **AI-generated + hand cleanup** (not commission / CC0 / self-drawn).
- Split: **code now + prompt kit** — the sprite-swap machinery lands and verifies this session;
  the art is generated by the founder off a kit I provide, and renders as PNGs land.

## 1. Constraint — original art only

Repo is **public under a personal name, MIT-licensed**. TPC actively DMCAs fan projects, so **no
ripped Pokémon / Game Freak / TPC assets**. The *style* (soft chibi pixel front-sprites) is not
protected; specific franchise creatures and ripped assets are. All 10 sprites are original,
generated from generic descriptors that never name a protected franchise.

## 2. Architecture — one component, one registry, emoji fallback

### Key correctness call: sprites live in the UI layer, NOT the engine

Plan 2's spec §8b sketched "add a `sprite` field to `Species`." **We deviate deliberately.**
`src/engine/` is pure / React-Native-free and is vitest-tested under a node environment. A
`require('....png')` inside `content.ts` (or a non-serializable `ImageSourcePropType` on the
`Species` type) would pull an RN/Metro asset into the pure engine and break the node test run.

Therefore: **no change to `Species`, `Creature`, `types.ts`, or `content.ts`.** The engine stays
pure. Sprite mapping lives entirely in a new UI-layer registry keyed by `SpeciesId`.

### New files

**`src/ui/sprites.ts`** — the registry.

```ts
import type { ImageSourcePropType } from 'react-native';
import type { SpeciesId } from '../engine/types';

// One line per landed PNG. A species absent here renders its emoji fallback.
// Keep every entry pointing at a file that EXISTS — Metro resolves require() at
// bundle time, so a require() of a missing asset breaks the build.
export const CREATURE_SPRITES: Partial<Record<SpeciesId, ImageSourcePropType>> = {
  // fernling: require('../../assets/creatures/fernling.png'),
  // pebblepup: require('../../assets/creatures/pebblepup.png'),
  // ...
};
```

Starts **empty** (every line commented). This is what keeps `main` green with zero art present.

**`src/ui/components/CreatureIcon.tsx`** — the single render path for a creature's face.

```tsx
type Props = { speciesId: SpeciesId; emoji: string; size: number };

export function CreatureIcon({ speciesId, emoji, size }: Props) {
  const sprite = CREATURE_SPRITES[speciesId];
  if (sprite) {
    return <Image source={sprite} style={{ width: size, height: size }} resizeMode="contain" />;
  }
  return <Text style={{ fontSize: size }}>{emoji}</Text>;
}
```

Both call sites have the data: `Creature.species` + `Creature.emoji`, and `Species.id` +
`Species.emoji`.

### Swap the three emoji sites

| Site | Current | New | Size |
|---|---|---|---|
| `CreatureRoster.tsx:34` | `<Text style={styles.emoji}>{c.emoji}</Text>` | `<CreatureIcon speciesId={c.species} emoji={c.emoji} size={24} />` | 24 |
| `DiscoveryToast.tsx:16` | `<Text style={styles.emoji}>{sp.emoji}</Text>` | `<CreatureIcon speciesId={sp.id} emoji={sp.emoji} size={56} />` | 56 |
| `DungeonCard.tsx:58` | `{c.emoji} {c.name}` (inline chip) | `<CreatureIcon speciesId={c.species} emoji={c.emoji} size={16} />` + `{c.name}` | 16 |

The DiscoveryToast reveal is the highest-value sprite moment — the hook of the whole loop.

Dungeon *definition* emoji (🍄/🌿/🌲 on `Dungeon`) and crop emoji stay emoji — this plan is
creatures only.

## 3. Result: incremental, never-blocked

With an empty registry, every site renders the emoji exactly as today. As the founder drops
`assets/creatures/<speciesId>.png` files and uncomments the matching registry line, each creature's
real sprite lights up independently. No half-broken commit, no all-or-nothing.

## 4. Art half — the prompt kit (deliverable)

Committed alongside this spec (in this file, §6). Contains:

- **Shared style anchor** — one reusable prompt prefix: soft cozy chibi forest critter, single
  front-facing character, flat pastel palette, thick clean outline, plain flat background, centered,
  pixel-art.
- **10 per-creature prompts** — Fernling, Pebble Pup, Moss Mouse, Bark Bug, Hedgehush, Cedar Cat,
  Lumi Fox, Owlin, Stagheart, Ember Kit — each a short generic descriptor appended to the anchor.
- **Negative / guardrail** — never the words "Pokémon", "Game Freak", "Nintendo", or any named
  franchise creature; no trademarked designs; generic animals + plants only.
- **Cleanup spec** — 64×64 px, transparent background (erase the flat bg after gen), consistent
  palette across the set, filename exactly `<speciesId>.png` (e.g. `fernling.png`), drop into
  `assets/creatures/`.

## 5. Testing

- **Engine tests untouched** — no engine change → the 62 vitest tests hold unchanged.
- **No new runtime test** — the registry type `Partial<Record<SpeciesId, ImageSourcePropType>>`
  already makes a typo key a `tsc` error, so a key-validity test is redundant. It would also be
  actively harmful: once real `require('....png')` lines exist, importing `sprites.ts` under
  vitest/node breaks (Metro resolves those assets, node can't). `tsc --noEmit` is the guard.
- **Manual mount (mandatory)** — live browser drive of all three screens with an empty registry
  (emoji shows), then wire one placeholder PNG and confirm the `<Image>` branch renders at the
  right size. This session's own gotcha: static review missed a zustand mount crash; only a real
  mount caught it. No component-test coverage exists, so a real mount is required, not optional.
